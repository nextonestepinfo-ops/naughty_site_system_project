# NOS OS 完成版 構築プロンプト v2

あなたはシニアプロダクトエンジニア兼デザインリードです。以下の仕様で、社内タスク管理＆AI秘書アプリ「NOS OS」を**完成品質**で構築してください。iOSアプリとしてストアに出せるレベルの体験を、まずPWAとして実現します。

-----

## 1. プロダクトの定義

- 名称: **NOS OS** — 「チームの今日を、ひとつの画面に」
- 対象: 4名の小規模チーム(管理者2名・社員2名)。スマホ(390px)での日常運用が主。
- 中核価値: ①開いた瞬間に「いま何をすべきか」が1件に絞られて見える ②AI秘書が段取り・整理・相談に即応する ③タスク・勤怠・通知が1アプリで完結する。

## 2. 技術スタック(変更不可)

- Next.js (App Router) + TypeScript + Tailwind CSS + React Query
- DB: Supabase PostgreSQL。データ層は `NOS_OS_DATA_MODE=mock | supabase` で切替できる Repo 抽象を必須とする(mockはメモリ内、キーなしで全機能デモ可能)
- AI: OpenAI API。**サーバーサイドのみ**で呼び出す。`OPENAI_API_KEY` 未設定時はローカル簡易ロジックにフォールバックし、レスポンスに `source: "openai" | "local"` を明示
- `SUPABASE_SERVICE_ROLE_KEY` / `OPENAI_API_KEY` をブラウザに露出させたら失格
- PWA: manifest + Service Worker + standalone + ショートカット(/tasks /assistant /attendance /notifications)
- APIレスポンスは常に `{"data": ...}`、エラーは `{"error":{"message": ...}}`

## 3. デザインシステム(v2 — 必ずこの通りに)

### トーン

「夜の司令室」。濃紺の落ち着きに、金色の”いまやること”が一点だけ灯る。AIは紫で別格扱い。装飾は最小限、余白と階層で品を出す。

### カラートークン

- ink(基調紺): 50 #F4F6FA / 100 #E9EDF4 / 200 #D2D9E6 / 300 #A6B1C8 / 400 #7587A8 / 500 #46587E / 700 #243353 / 800 #16203B / 900 #0B1226 / 950 #070D1D
- gold(最優先・行動): 300 #F7C878 / 400 #F0A93C / 500 #E08F12
- iris/violet(AI専用): #818CF8 / #6366F1 / #8B5CF6
- semantic: green #10B981 / red #EF4444 / orange #F59E0B

### タイポグラフィ

- フォント: `-apple-system, BlinkMacSystemFont, "Hiragino Sans", "Noto Sans JP", sans-serif`(iOSネイティブ感はシステムフォントで出す)
- 各画面はiOS式ラージタイトル: 12pxの英字キッカー(letter-spacing .14em、ink-400) + 30px/800のタイトル(letter-spacing -.02em)
- 数値は `font-variant-numeric: tabular-nums`
- 入力要素は16px以上(iOSズーム防止)

### コンポーネント言語

- カード: 白 / radius 20px / 影 `0 1px 2px rgba(11,18,38,.05), 0 8px 24px -10px rgba(11,18,38,.12)` / 1pxのほぼ白ボーダー
- ボタン: radius 16px / font-weight 700 / `:active` で scale(.96)、イージングは `cubic-bezier(.34,1.4,.4,1)`(ばね感)
- セグメント切替: iOS式(灰背景+白サムが滑る)。チップは pill 形
- シート: 下から `cubic-bezier(.34,1.4,.4,1)` でスライドイン、上部にグラバー、背景は暗転+blur
- タブバー: `backdrop-filter: blur(20px) saturate(1.6)` のすりガラス、上1pxヘアライン、safe-area対応
- トースト: 下部に濃紺pill、2.2秒で消える
- `prefers-reduced-motion` でアニメ全停止

### シグネチャー(この2つがNOS OSの顔)

1. **司令室ヒーロー**: ホーム最上部。濃紺グラデ(#101A36→#0B1226、右上に紫・左下に金のradial光)のカードに、最優先タスク1件+conic-gradientの金色スコアリング(優先度スコアを0-100で表示)+「開始する」(金グラデ)/「秘書に相談」(ガラス)ボタン。金ドットがpulseする「いまの最優先」アイキャッチ付き。
2. **AIオーブ**: タブバー中央。58pxの紫グラデ円が-26px浮き、白3pxリング+紫の発光影。タブは ホーム/タスク/[オーブ:秘書]/勤怠/通知 の5つ。

## 4. 画面仕様

- **ログイン**: 濃紺フルスクリーン。メンバー4名の顔グリッド(イニシャルアバター、各員固有グラデ)→ iOS式パスコード入力(ドット4つ+円形キーパッド、誤入力でshake)。初期パスワード「0000」、初回ログイン時はパスワード変更を強制。
- **ホーム(今日の司令室)**: 挨拶(時間帯で変化)+日付キッカー / ヒーロー / 統計4タイル(今日・超過※赤・進行中・完了) / 週ストリップ(今日は紺反転、タスク有りの日に金pip) / このあとのタスク3件 / チームの今(各メンバーの状態+残件数、管理者には勤怠状況)。
- **タスク**: セグメント(今日/今週/すべて)+チップ(全部/自分の担当/優先度高/進行中/保留)。社員はデフォルト「自分の担当」。タスク行=丸チェック(完了で✓→縮小フェードアニメ)+タイトル+バッジ列(【テスト】/優先度/状態/期限※超過は赤・今日は金)。タップで詳細シート(開始/確認待ち/保留/カレンダー/完了/削除※confirm必須)。金色FABで追加シート。一覧下部に「AIにタスクを整理してもらう」。
- **秘書(AI)**: フルハイトチャット。濃紺=自分/白カード=AI、登場は下から8pxフェード。タイピングインジケーター(3ドット)。サジェストチップ(段取り/整理/振り返り/明日の準備)。整理提案は紫淡グラデの**提案カード**として表示し「反映する/見送る」を1件ずつ。反映後は「✓ 保留に変更しました」に変わる。
- **勤怠**: 54pxライブ時計+状態pill(勤務中=緑ドット等)。打刻8種(出勤/退勤は2カラム大、休憩入り/戻り・外出/帰社・打合せ/欠勤)。当日のタイムライン(金ドット+ヘアライン)。
- **通知**: 統計4タイル(本日期限/明日期限/超過※赤/未読)+通知リスト(重要度カラードット、既読で半透明)。タブバーに未読バッジ。cronで期限通知を生成(`NOS_OS_CRON_SECRET` で保護可)。
- その他: 案件CRUD(管理者のみ編集)、社員一覧/詳細、顧客一覧、ガイド、テスト(ヘルスチェック表示)、設定(パスワード変更/ログアウト)。

## 5. AI安全ルール(最重要・テストで検証すること)

1. AIはDBを直接変更しない。**常に提案のみ**を返し、ユーザーが1件ずつ反映する。
2. 「減らして」「整理して」→ **保留(hold)への変更提案のみ**。削除提案は出さない。
3. 削除提案は、ユーザーが明示的に「削除」と言った場合のみ。反映時も1件ずつ個別confirm。
4. 「削除以外を反映」一括ボタンを提供(create/updateのみ適用)。
5. 整理対象のブランチ(目標ツリー)が指定されている場合、AIの返した projectId よりブランチ側を優先。

## 6. データ・初期値

- ユーザー: 浦田 和真(admin)/ 大崎 雄介(admin)/ 橋迫 翔太(employee)/ 渡邉 駿(employee)。全員初期PW「0000」+ must_change_password=true。
- 権限: 管理者は全件閲覧・全操作、社員は自分の担当のみ(UI上)。
- テストデータは全タイトルに**【テスト】プレフィックス**+UIでテストバッジ表示。顧客2件・案件3件・目標ツリー1本(ブランチ2)・タスク6件(今日締切2・超過1を含む)・勤怠・休暇。
- seed は決定論的UUID+upsertで**非破壊**(既存ユーザーのパスワードを上書きしない)。`npm run seed` で実行。
- 優先度スコア = 期限0.4 + 優先度0.3 + 顧客待ち0.2 + 遅延リスク0.1(0-100)。

## 7. カレンダー連携(片方向)

- タスク→Googleカレンダー: `calendar.google.com/calendar/render?action=TEMPLATE` リンクに302リダイレクトするAPI。
- ICS: 全体 `/api/calendar/ics` とタスク単体のICSエンドポイント(text/calendar)。

## 8. 完成条件(これを満たすまで終わらない)

1. `tsc --noEmit` と `next build` がクリーンに通る。
2. mockモードのスモークテストが全通過: ログイン(誤PW 401/未認証 401)・初回PW変更→再ログインでmustChange解除・ダッシュボード・タスクCRUD+フィルタ・AI整理(「整理して」→hold提案のみ/delete 0件、source明示)・秘書チャット・打刻・ICS 200・Googleリンク302・cron通知生成→担当者に表示→既読化・全ページ200。
