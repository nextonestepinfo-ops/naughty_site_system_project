# 管理システム作業場所

ここに、今後ローカルで動く管理システムを作る。

## 最初に作る画面案

- ログイン
- 管理者トップ
- スタッフ管理
- スタッフ写真/紹介文編集
- 2週間シフト管理
- 出退勤/勤務ステータス
- 商品マスター
- 売上登録
- CSV出力
- 給与確認
- 従業員アカウント管理
- ギャラリー公開管理

## Webサイト連動

管理システム側のデータを、Webサイト側が読み込める形式で出す。

最初はJSONでもよい。

## データ保存とバックアップ

- 現行CMSデータはブラウザ内の `localStorage` キー `naughty.siteData.v1` に保存される。
- 外部DBではないため、別端末・別ブラウザには自動同期されない。
- ブラウザのデータ削除、シークレットモード、端末故障、別端末操作ではデータが見えなくなる可能性がある。
- 管理画面の「バックアップ」でJSONを書き出し、「復元」で読み戻せる。
- 保存時には `naughty.siteData.backups.v1` に最大5世代の自動バックアップ履歴を残す。
- 複数端末で完全に共有するには、将来的にサーバーDBまたはGitHub/Sheets等への同期APIが必要。

## 現在のプロトタイプ

- `index.html`
  - スマホ運用を想定したローカル管理画面。
  - ホーム、スタッフ、従業員アカウント、ギャラリー、シフト、メニュー、イベント、売上、給与の初期画面を実装。
- `app.js`
  - `../03_system_seed/naughty_site_data.json` を初期値として読み込む。
  - 保存時に `localStorage` へ反映し、同じローカルURLの公開サイトが読み込めるようにする。
  - ログインID方式の従業員アカウント、QR勤怠の実打刻/15分切り捨て時刻、売上CSVプレビュー、勤怠CSVを含む。
- `styles.css`
  - 大きいボタン、選択式、少ない入力を優先した管理UI。
- `../06_employee_portal/index.html`
  - 従業員用のログイン、QR打刻、注文伝票、自分の売上、プロフィール画面。
- `../supabase/`
  - 本番移行用のスキーマとEdge Function雛形。

ローカル確認時はプロジェクトルートでHTTPサーバーを起動し、`/05_admin_system/index.html` を開く。
