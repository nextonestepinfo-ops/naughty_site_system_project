# 進める順番

## 1. 既存Webサイト案の整理

- `01_existing_site/` の既存デザインを確認する。
- トップ部分で必ず残す要素を決める。
- 素材のうち、使ってよいもの/差し替えるものを分ける。
- Instagramを参考に、実店舗の雰囲気に近い見せ方へ寄せる。

## 2. 共通データ設計

Webサイトと管理システムで共通して使う情報を先に決める。

- スタッフ
- スタッフ写真
- スタッフ紹介文
- 出勤予定
- 勤務ステータス
- 商品/メニュー
- イベント
- 店舗情報

## 3. 管理システムのMVP

最初はテストデータ入りで作る。

- ログイン
- スタッフ管理
- スタッフ写真/紹介文管理
- 商品マスター
- シフト管理
- 出退勤
- 売上登録
- 売上CSV出力
- 給与計算の土台

## 4. Webサイトの再構成

- トップは既存案の良い部分を活かす。
- スタッフ紹介、出勤、シフト、イベント、メニューは管理データから表示できる形にする。
- スマホ表示を優先して見直す。

## 5. 連動確認

管理画面で更新した内容がWebサイトに反映されるか確認する。

- スタッフ写真を変える
- 紹介文を変える
- 出勤予定を変える
- イベントを追加する
- メニューを追加する

## 6. 後回しでよいもの

- 本格的なサイトエディター
- 細かいレイアウト編集機能
- ICカード打刻
- 給与明細PDF
- 複雑なイベント歩合条件
- 高度な権限設計
