# Webサイトと管理システムの連動モデル

## 基本方針

管理システム側を「情報の入力場所」にする。  
Webサイト側は、その情報を読み込んで表示する。

管理者がスマホで触る画面は、Webサイトを直接デザイン編集する画面ではなく、次のような簡単な更新画面にする。

- スタッフを追加する
- 写真を登録する
- 紹介文を書く
- 今日出勤にする
- 2週間シフトを更新する
- イベントを追加する
- メニューを追加する
- 表示/非表示を切り替える

## 反映ルール案

| 管理システム側 | Webサイト側 |
|---|---|
| スタッフマスター | スタッフ紹介一覧 |
| スタッフ写真 | スタッフカードの写真 |
| 紹介文 | スタッフ詳細/カード本文 |
| 勤務ステータス | 本日の出勤/出勤中表示 |
| 2週間シフト | シフト表 |
| イベントマスター | イベント一覧 |
| 商品マスター | メニュー一覧 |
| 店舗情報 | アクセス/営業時間 |

## 画面イメージ

### 管理者トップ

- 今日の出勤
- スタッフ更新
- シフト更新
- イベント更新
- メニュー更新
- 売上CSV
- 給与確認

### スタッフ編集

- 写真
- 表示名
- 紹介文
- SNSリンク
- 出勤表示する/しない
- Webサイトに表示する/しない

### シフト編集

- 今日
- 明日
- 2週間分
- スタッフごとの出勤/休み/未定
- 表示用コメント

## 注意点

- Webサイトに出す写真は、登録時にサイズを揃える必要がある。
- 紹介文は長すぎるとスマホで読みにくいので、短文欄を基本にする。
- 管理画面の自由度を高くしすぎると、スマホ運用が難しくなる。
- まずは「情報を入れたら決まったデザインに反映される」形にする。
