# スコープと方向性

## Webサイト側

### 必須で活かすもの

- 既存Webサイト案のトップ部分。
- Vすぽ系を参考にした、動き・奥行き・ロゴ/キャストの見せ方。
- ノーティの黒、ピンク、白を使った世界観。
- 小悪魔に寄せすぎず、かわいい寄り。

### 再検討するもの

- トップ以外のページ構成。
- 素材の使い方。
- スタッフ紹介の見せ方。
- シフト/出勤情報の見せ方。
- イベント、メニュー、募集ページの構成。
- スマホ表示の読みやすさ。

### サイトで表示したい情報

- 店舗トップ
- スタッフ紹介
- 本日の出勤
- 2週間シフト
- イベント
- メニュー
- キャスト募集
- アクセス
- InstagramなどSNS導線

## システム側

### 管理したい情報

- スタッフマスター
- スタッフ写真
- スタッフ紹介文
- 勤務ステータス
- 出勤/退勤
- 2週間シフト
- 商品マスター
- 販売金額
- バック金額
- イベント歩合
- 売上CSV出力
- 日給計算
- 給与明細

### サイト連動の考え方

本格的なWebサイトエディターではなく、入力済みの店舗データをWebサイトに反映させる。

例:

- スタッフマスターの写真、名前、紹介文をWebサイトのスタッフ紹介に表示する。
- 出勤予定をWebサイトの「本日の出勤」「シフト」に表示する。
- イベントマスターをWebサイトのイベント欄に表示する。
- 商品マスターをメニュー欄に表示する。
- 勤務ステータスを必要に応じてWeb側に出す。

## 操作性の方針

- スマホで使えることを優先する。
- 管理者画面は大きいボタンと選択式を中心にする。
- 文章編集は、長い自由入力よりも「紹介文」「一言コメント」「表示/非表示」などに分ける。
- 難しいデザイン編集はさせない。
- 写真登録、紹介文更新、シフト更新、イベント更新だけできればよい。
